# Guia da Escada de Valor (Value Ladder)

Framework para estruturar portfólio de produtos do mais barato ao mais caro, maximizando LTV.

## Conceito

A escada de valor é a sequência de produtos/serviços ordenados por preço e valor entregue. Cada degrau resolve um problema maior ou entrega um resultado mais profundo. O cliente sobe naturalmente de um degrau para o próximo conforme ganha confiança e obtém resultados.

**Princípio central**: Nunca pedir para o cliente dar um salto de fé grande demais. Cada degrau precisa ser acessível a partir do anterior.

## As 4 Alavancas de LTV

### 1. Vender mais do mesmo
- Recompra, renovação, recorrência
- Exemplo: assinatura mensal, consumíveis, créditos
- Ação: garantir que o cliente extraia valor para renovar

### 2. Vender algo diferente (upsell/cross-sell)
- Próximo degrau da escada ou produto complementar
- Exemplo: do plano básico para o premium, adicionar módulo
- Ação: mapear o momento certo de oferecer o próximo degrau

### 3. Casos de uso correlacionados
- Resolver problemas adjacentes do mesmo cliente
- Exemplo: empresa que vende CRM também vende automação de marketing
- Ação: identificar "o que mais esse cliente precisa resolver?"

### 4. Lembrar que você existe (reativação)
- Clientes inativos, churned, ou que nunca compraram
- Exemplo: campanha de reengajamento, oferta especial de retorno
- Ação: manter relacionamento mesmo quando não está vendendo (conteúdo, comunidade)

## Como Identificar Gaps na Escada

### Regra do 3x
Se o salto de preço entre dois degraus é maior que 3x, provavelmente falta um degrau intermediário.

**Exemplo de gap**:
- Produto A: R$500
- Produto B: R$5.000 (10x de salto — falta algo entre R$1.500 e R$2.500)

### Sinais de gap na escada
- Taxa de conversão entre degraus < 10% (o salto é grande demais)
- Clientes dizem "é caro" para o próximo degrau (falta degrau intermediário)
- Clientes satisfeitos não têm para onde ir (falta degrau superior)
- Muitos leads mas poucos compradores (falta degrau de entrada acessível)
- Clientes compram uma vez e somem (falta recorrência ou próximo passo claro)

| Tipo | Função | Margem Típica | Exemplo |
|------|--------|---------------|---------|

## Escadas Comuns por Indústria

### SaaS B2B
1. Free trial / Freemium → 2. Plano básico (self-service) → 3. Plano profissional → 4. Plano enterprise (custom) → 5. Consultoria de implantação + suporte dedicado

### E-commerce
1. Produto de entrada (baixo risco) → 2. Kit/combo (ticket médio maior) → 3. Assinatura/clube → 4. Linha premium/exclusiva → 5. Experiência personalizada

### Educação/Infoprodutos
1. Conteúdo gratuito → 2. Curso online gravado → 3. Programa ao vivo/cohort → 4. Mentoria em grupo → 5. Mentoria individual/mastermind

### Serviços Profissionais
1. Diagnóstico gratuito → 2. Projeto pontual → 3. Retainer mensal → 4. Consultoria estratégica → 5. Advisory/board seat

## Psicologia de Preço: Por Que os Gaps Importam

### Ancoragem
O cliente avalia preço relativamente ao que já pagou. R$2.000 parece razoável depois de pagar R$997, mas parece caro se o último contato foi um conteúdo gratuito.

### Confiança incremental
Cada compra bem-sucedida aumenta a confiança para a próxima. O cliente precisa de vitórias pequenas antes de investimentos grandes.

### O efeito "degrau ausente"
Quando falta um degrau, o cliente para de subir. Não é que ele não quer o próximo nível — é que o salto parece arriscado demais. Inserir um degrau intermediário frequentemente desbloqueia receita que já existia latente.

### Preço como filtro
Cada degrau filtra naturalmente o público. Nem todo cliente precisa (ou deveria) chegar ao topo. A escada garante que cada perfil encontra o degrau adequado.

## Checklist de Diagnóstico

- [ ] Tenho produto de entrada acessível (baixo risco para o cliente)?
- [ ] Nenhum salto de preço é maior que 3x?
- [ ] Cada degrau tem um call-to-action claro para o próximo?
- [ ] Meço taxa de conversão entre degraus?
- [ ] Tenho mecanismo de recorrência (assinatura, renovação, recompra)?
- [ ] Clientes satisfeitos no topo têm para onde ir (comunidade, exclusividade)?
- [ ] Clientes que saíram têm caminho de retorno (reativação)?
